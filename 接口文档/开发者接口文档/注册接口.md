

## 用户注册


**接口地址**:`/specify/sys/user/register`


**请求方式**:`POST`


**请求数据类型**:`application/x-www-form-urlencoded,application/json`


**响应数据类型**:`*/*`


**接口描述**:<p>匿名注册接口（Shiro 已放行 /sys/user/register）。请求体为 JSON，字段如下：</p>
<ul>
<li><code>phone</code> (String, 必填): 手机号</li>
<li><code>smscode</code> (String, 必填): 短信验证码</li>
<li><code>password</code> (String, 可选): 登录密码，留空则随机生成 8 位</li>
<li><code>username</code> (String, 可选): 登录账号，留空则以手机号作为用户名</li>
<li><code>email</code> (String, 可选): 邮箱</li>
<li><code>realname</code> (String, 可选): 真实姓名，留空则同 username</li>
</ul>
<p>短信验证码来源：</p>
<ul>
<li>正式流程需先调用 <code>POST /sys/sms</code>（smsmode=1）获取并下发验证码，600s 内有效。</li>
<li>仅 dev/test 环境：若配置了 <code>jeecg.firewall.smsMockCode</code>（如 666666），该&quot;万能码&quot;可直接通过校验，
无需真实下发；真实验证码仍然优先生效。生产环境配置该项会导致应用启动失败。</li>
</ul>
<p>校验顺序：用户名/手机号/邮箱查重 → 短信验证码校验。注册成功不分配角色、不自动登录，需另行登录。</p>
<p>业务错误以 HTTP 200 + <code>success=false</code> + <code>message</code> 返回（此接口沿用旧式响应，未使用 Result.code 错误码）。</p>



**请求示例**:


```javascript
{
  "phone": "",
  "smscode": "",
  "password": "",
  "username": "",
  "email": "",
  "realname": ""
}
```
