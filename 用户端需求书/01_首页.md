# 01 首页 (Home)

> 核心理念：极简、有温度的个人数字陈列室。通过区块分割优雅地容纳创作与消费双重属性，用模板与渐进披露打造极致 C 端体验。前置要求：强制开发者登录后才可进入此页面。

---

## 一、 页面整体布局 (Personal Gallery)

采用大留白、去边框的现代画廊（Gallery）布局，营造专注、沉浸的浏览体验。

### 1.1 极简顶部导航
去掉一切冗杂的按钮。
*   **左侧**：产品 Logo。
*   **右侧**：极简的用户头像。点击头像弹出全局设置下拉菜单，包含：
    *   `API 密钥配置` (集中管理 API 凭证池的专门入口，详见下文 3.0 说明)
    *   `账号设置`
    *   `退出登录`

### 1.2 动态微引导 (Dynamic Banner)
位于页面顶部中央区域，打破静态页面的呆板感。用极其克制的单行文字，在不打扰开发者的前提下提供情感价值或使用技巧。
*   **示例 (问候)**："👋 下午好，张三。开始构建你的专属智能体吧。"
*   **示例 (Tips)**："💡 Tip: 在开发页的指令中输入 @ 可以快速引用文件库。"

---

## 二、 智能体陈列区 (上下区块分割 Sections)

画廊在垂直方向上通过极其轻量的文字标题（无生硬分割线或侧边栏）划分为不同的区块 (Sections)。

### 2.1 区块一：我的专属创作
展示开发者自己创建的 Agent 以及新建入口。

```text
  我的创作
  
  ┌──────────────┐   ┌──────────────┐   ┌──────────────┐
  │      ➕      │   │ 🤖 translator │   │ 📝 report_gen  │
  │ 从零开始创建   │   │              │   │              │
  └──────────────┘   └──────────────┘   └──────────────┘
```

*   **卡片 1：从零开始创建**：常驻首位，大尺寸虚线边框卡片。
*   **自建卡片展示：渐进披露 (Hover Reveal)**：
    *   **常态（静默）**：为了保持画廊的呼吸感，卡片在日常状态下绝对不显示任何多余的按钮，只显示 **App名称**、**创建时间** 和 **调用的工具数**。
    *   **悬浮 (Hover)**：当鼠标悬浮时，卡片底部平滑滑出操作面板：`[▶ 运行对话]` (高亮为主操作) | `[⚙️ 编辑]` | `[🔗 分享]` | `[🗑 删除]`。

### 2.2 区块二：最近使用 (他人分享)
展示开发者通过他人分享的链接运行过的非自己创建的 Agent。

```text
  最近使用
  
  ┌──────────────┐   ┌──────────────┐
  │ 📊 data_bot   │   │ 🎨 poster_gen │
  │ (来自: 李四)  │   │ (来自: 王五)  │
  └──────────────┘   └──────────────┘
```

*   **来源标识**：卡片描述下方用浅灰色显示创建者信息（如 `来自: 李四`），强化“这是他人的工具”心智。
*   **操作差异**：悬浮 (Hover) 时仅滑出 `[▶ 运行对话]` 和 `[🗑 从记录中移除]`。无编辑和分享权限。

## 三、 核心操作与边界链路

### 3.0 API Key 延迟验证机制 (留存用户)
为了最大限度降低新开发者门槛并留住开发者，系统采用**延迟验证机制**：
*   **浏览与创建阶段不设卡**：开发者进入首页、创建App、编辑提示词、组合工具等前期操作，均**不强制**要求填写 API Key。
*   **使用时拦截 (Just-in-Time)**：只有当开发者第一次点击卡片上的【▶ 运行对话】或者在开发页点击运行测试时，如果系统检测到尚未配置对应大模型的 API Key，才会在当前页**居中弹出一个模态框（Modal）**要求输入 API Key。填写完毕后，自动继续原本的运行流程。
*   **统一管理池 (API 密钥配置)**：
    *   拦截弹窗填写的 Key 会自动保存到全局设置的「API 密钥配置」资源池中。
    *   **管理形态**：点击头像下的 `[API 密钥配置]` 时，在首页居中弹出一个宽大的管理模态框。
    *   **展示逻辑**：采用“极简大列表”平铺（不按模型强行分类，以兼容多模型中转 API）。
    *   **核心字段**：每行仅展示四大核心字段：`[备注别名]` | `[Base URL]` | `[API Key (掩码)]` | `[操作 (编辑/删除)]`。像一个纯粹的密码本一样干净高效。

### 3.1 从零创建：跳转缓冲页
点击“从零开始创建”卡片后，**进入创建App缓冲页**，以降低新手的认知压力，实现平滑转场。
在缓冲页中：
1. 采用全屏留白设计，让开发者专注填写基础信息：**「App 名称」**（校验规则：仅支持全英文+下划线）、**「App 描述」**。
2. 并在同页面下方挑选**「底层模型大脑」**。
3. 点击“创建并进入”后（*依然不强求填 API Key*），带着这个设定的身份平滑转场进入 `02 开发页面`。

### 3.2 零数据破局：模板启动法 (Templates)
当新注册开发者首次进入，没有任何卡片数据时，页面不能过于空旷。
*   **空状态提示**：提供温馨的空状态文案引导，如：*“看起来你还没有创建任何应用，试试从模板开始，或者从零开始构建吧！”*
*   **官方优质模板**：在“我的创作”区块，自动并列展示 3 个官方推荐的模板卡片（例如：`doc_analyzer`、`web_summarizer`），并在视觉上加以区分（如增加 `[💡 官方模板]` 角标）。
*   **差异化的交互逻辑 (一键创建)**：
    *   **正常自建 App 卡片**：开发者点击日常的自建卡片，默认行为是进入 `05_运行页面_调试模式` 或 `02_App开发页面` 进行对话和编辑。
    *   **官方模板卡片**：开发者点击模板卡片，走的是**“快捷创建 App”**逻辑。系统会在后台一键克隆该模板的所有预设配置，直接生成一个归属于当前开发者的新 App 实例，并立即跳转进入该新 App 的 `02_App开发页面`。开发者随后可以直接在该页面进行二次定制或调试。

## 四、 非功能性体验与边缘场景

**4.1 加载态与过渡 (Loading)**
*   首页数据加载不使用整页转圈，统一采用 **骨架屏 (Skeleton)**。提前渲染出卡片的灰色轮廓占位，保证画廊风格的平滑过渡感。

**4.2 列表渲染与加载 (Pagination)**
*   **加载策略**：不使用传统分页控件。首次加载优先渲染约 20 个卡片（填满首屏视野），向下滚动时采用**瀑布流/触底自动加载更多 (Infinite Scroll)** 形式。

**4.3 异常数据容错机制**
*   **数据异常/封禁态**：若某 App 因数据请求失败或违规被拉黑等异常状态，卡片仍需展示，但要在显著位置（如右上角）增加红色 `[异常/封禁]` 状态标签。悬浮时，隐藏 `[▶ 运行对话]` 和 `[🔗 分享]` 按钮，仅保留 `[⚙️ 编辑]` 和 `[🗑 删除]`。

**4.4 文本展示与截断规则**
*   **App 名称截断**：卡片上的 App 名称**至多显示 1 行，超出部分以省略号 `...` 截断**。
*   **全名提示**：当名称被截断时，鼠标 Hover 到名称上方需出现轻量级的 `Tooltip` 浮窗显示完整名称。

**4.5 模块按需隐藏 (防御性空状态)**
*   “最近使用”区块为按需展示。当系统判定开发者没有任何“使用他人分享App”的历史记录时，该区块连同标题一起**完全隐藏**，保持页面纯净。

**4.6 卡片状态透出 (发布标识)**
*   首页卡片保持极简，不承载复杂的版本管理操作（复杂的版本流转和发布管理归属于【02_App开发页面】）。
*   如果某 App 已经有成功分享/发布上线的版本，可在卡片名称旁增加极简的 `[🌐 已发布]` 徽标作为轻量级状态透出，赋予开发者成就感。

*(注：当开发者未产生任何使用他人 Agent 的记录时，“最近使用”区块会自动隐藏，保持页面绝对纯净。)*
