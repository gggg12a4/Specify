# 方案4:出错 Agent 与引用失效的处理(在 02 真实布局上叠加)

> 沿用 `02_App开发页面_new.md §一` 真实布局,只叠加失效态。本方案整合本轮讨论定稿的全部规则与交互。

---

## 一、核心规则(5 条)

1. **出错的 Agent**(直接引用了失效工具)不能处于启用状态,**只有修复后才能启用**。
2. 出错的 Agent 出路:**修复** 或 **删除**;处理完之前 **App 保存不了**(阻断)。
3. **删除 Agent 时,级联删除所有引用它的地方**(删 B → A 里对 B 的引用一并删)。
4. 或者:**在引用方里取消对它的引用**(A 不再用 B)。
5. **未启用的 Agent 若内部有工具失效,点"启用"时提醒"请先修复",不让直接启用**。

→ 出错的 Agent 进不了启用链;**全程不递归**,链上每个 Agent 各报自己直接引用的失效项。

## 二、行首符号三态

| 状态 | 行首 |
|---|---|
| 启用且正常 | `☑` |
| 未启用(正常待选) | `☐` |
| **启用中但失效/出错** | `⚠️`(替代勾选框) |

**展开的子工具失效图标**(纯展示):`❌` = 平台工具失效 · `⚠️` = 引用的 Agent 工具失效 · 正常子工具纯名字。

## 三、列表展开规则

- **每个 Agent(不分正常/出错)行下,横向展开它引用的全部子工具**(正常+失效都列,chips 式一行,放不下自动换行)。
- 子工具**纯展示**:无勾选框、无操作按钮(启用/修复都在编辑弹窗里做)。
- 失效子工具只用**图标**区分,不写长文案:`❌`=平台工具失效 · `⚠️`=引用的 Agent 工具失效 · 正常子工具纯名字。
- 外层 Agent 行保留其**正常描述**(不堆"引用的工具已失效"这类红字,免得满屏吓人)。
- 行尾按钮:**内部含失效引用**的 Agent =「修复」(不管启没启用);内部全正常 =「编辑」。

数据:主AI 启用 a(读取文件)b(绘图)c(图片理解,未开)m1(notion);A(子=a,B)、B(子=a,c,C)、C(子=a)、**E(子=b,c,全正常)** 均启用;D(子=a) 未启用。**a 失效下架**。其中 **E 不引用 a,作为"完全正常的 Agent"对照**。

读图:
- 推荐工具区 a:启用中失效 → 行首 `⚠️`,`[关闭]`(直接移除主 AI 的 a)。
- A/B/C:都直接引用了 a → 各自 `⚠️` 出错,行尾 `[修复]`,**各报各的**(A 不替 B 报)。
- 每个 Agent 行下**横排**展开全部子工具(纯展示,无勾选无按钮);失效只用图标:`❌`=平台工具失效、`⚠️`=Agent 工具失效,正常子工具(如 c)纯名字、用 `·` 分隔。
- **E 启用且子工具(b、c)全正常 → 行首 `☑` + `[编辑]`,子工具横排纯名字、无任何图标**(完全正常的 Agent 长这样,作对照)。
- D 未启用 → `☐` + `[修复]`(它内部有失效的 a);子工具照样横排展开,失效的 a 标「未启用,暂不影响」;开发者点启用 D 时提醒「请先修复」。
- 顶部全局警告条 + `[保存]` 置灰,直到所有启用链上的失效都处理完。

## 五、详情弹窗(点 AgentA 的 [修复] 进入)

> A 的子工具 = a(平台工具,失效) + B(Agent 工具,失效)。两类失效给不同处理。

```
┌──────────────────────────────────────────────────────────────┐
│  修复工具：AgentA                                      [✕]   │
│  ────────────────────────────────────────────────────────    │
│  ┌─App资料─┐  ┌─工具配置──────────────────────────────────┐  │
│  │ ▼shared/│  │ 名称 [ AgentA ]  (只读)                    │  │
│  │  ├手册  │  │ 展示名称* [ …… ]                           │  │
│  │  └...   │  │ 系统提示词* [ …… ]                         │  │
│  │         │  │ 可用工具                                   │  │
│  │         │  │ ┌────────────────────────────────────────┐│  │
│  │         │  │ │ 推荐工具                                ││  │
│  │         │  │ │ ❌ 读取文件(a) 平台工具已下架   [关闭]  ││  │ ← 平台工具失效 → [关闭]
│  │         │  │ │ ☐ 绘图(b)                      [⚙️]    ││  │
│  │         │  │ │ 你创建的工具                            ││  │
│  │         │  │ │ ☑ AgentB  ⚠️（点击看建议）            ││  │ ← Agent工具失效:勾选框(取消=移除引用)+⚠️点击看说明
│  │         │  │ │ ☐ AgentC                              ││  │
│  │         │  │ │ MCP 服务                               ││  │
│  │         │  │ │ ☐ notion                              ││  │
│  │         │  │ └────────────────────────────────────────┘│  │
│  │         │  │                  [取消]   [保存]           │  │
│  └─────────┘  └────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────┘
```

- **平台工具失效(a)** → `❌` + `[关闭]`:A 直接引用的平台工具失效,**强制**处理(不处理 A 一直阻断保存),点 `[关闭]` 移除。
- **Agent 工具失效(B)** → 保留勾选框 `☑`(取消勾选 = 移除引用)+ 旁边 `⚠️` 提醒图标,**不强制**(B 失效会由 B 自己那行卡保存)。`⚠️` 默认不展开文案,**点击才弹出**「建议修改该创建的工具,或移除引用」。开发者可取消勾选移除引用,也可保留(改天去把 B 改好)。

---

## 六、处理与保存

- 修复一个 Agent 的所有失效项(平台工具 [关闭] / Agent 引用 [移除引用] / 或去把被引用 Agent 改好)→ 它从 `⚠️` 恢复 `☑`。
- 也可直接 `[🗑]` 删除该 Agent(确认弹窗提示「该工具被 X 引用,删除将一并移除引用」,级联清理)。
- 所有启用链上的失效都处理完 → 警告条消失,`[保存为草稿去调试]` 恢复。
- 未启用的 D 全程不阻断;点启用时若仍有失效,提醒「请先修复」。
