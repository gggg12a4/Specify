# 05 运行页面 — 调试模式交互重构

> 入口：在 App 开发页面点击 [▶ 进入调试]

---

## 一、 整体架构：原地翻转与沉浸式联动

通过**“原地翻转画布”**实现沉浸式沙盒体验；将技术细节（JSON、报错、拦截）封装在极简的**自然语言拟人化**与**渐进披露抽屉**之中。

1. **触发动作**：用户在 App 开发页面点击 `[▶ 进入调试]`。
2. **临门一脚的鉴权拦截 (Just-in-Time Auth)**：
   *   系统在点击 `[▶ 进入调试]` 时，校验当前 App 是否已绑定了所需大模型的 API 凭证。如果未绑定，将进行强制拦截配置。
   *   **白底极简模态框 (Modal)** 视觉呈现与交互：
       *   **文案引导**：“为了启动沙盒，请为该应用配置 [当前模型名称] 的 API 凭证。”
       *   **输入表单（支持第三方中转与资源池）**：
           1.  **凭证选择下拉框**：默认展示 `[+ 新增配置]`。如果用户的全局“密钥管理池”中已有同平台的历史记录（如“公司代理”、“个人直连”），可直接下拉选用，免去重复填写。
           2.  **API Base URL**：明文输入框。默认预填官方地址。开发者可自由修改为第三方中转站的 API 地址。
           3.  **API Key**：密码输入框。输入时掩码显示 `***`，尾部提供小眼睛图标可切换为明文。
   *   **数据流转与作用域（App 独立绑定）**：
       *   点击 `[保存并继续]` 后，这套配置**仅绑定至当前这个具体的 App**。其他 App 互不干扰，实现了彻底的计费和请求通道隔离。
       *   如果是手工 `[+ 新增配置]` 填写的，系统会在后台静默将这条新凭证存入首页头像下的**「API 密钥配置（管理池）」**中，方便下次创建其他 App 时直接下拉复用。
3. **布局翻转 (Canvas Swap)**：
   *   **左侧保持不动**：`资源与沙盒 (文件树)` 面板岿然不动。
   *   **中右合并**：原本中央的`系统提示词画布`与右侧的`工具配置抽屉`瞬间发生三维翻转或平滑横移。
   *   **呈现结果**：中右区域合并成为一个宽大纯净的**调试聊天画布 (Playground)**。
4. **沙盒生命周期 (Session Lifecycle)**：
   *   **对话阅后即焚**：调试沙盒的聊天记录仅为当前验证服务。点击顶部【清空对话】或退出重进时，对话流彻底清空，不留多余的历史会话列表。
   *   **文件状态解耦**：清空对话**不会**自动清空左侧 `workspace/`、`temp/` 等沙盒中的文件。这种解耦设计允许开发者测试“AI 对历史生成文件的跨轮次理解”。如需纯净环境，开发者需手动在左侧树中删除文件。
4. **空间联动感 (Spatial Connection)**：
   *   当 AI 在聊天框内读取、修改文件时，左侧文件树中对应的节点会有微弱的**高亮呼吸动效**。
   *   用户在右侧随时点击某个文件名引用，左侧树会自动展开并定位到该文件。

```text
┌───────────────────────────────────────────────────────────────────────────────┐
│ ← 首页 | App：cheesy_words_bot                             [结束调试] [发布版本]│
├─────────────┬─────────────────────────────────────────────────────────────────┤
│ ┌─左侧面板─┐│ 💬 调试沙盒 (Playground)                                        │
│ │ 📁 知识库││                                                                 │
│ │ ──────── ││  👤 帮我搜索 shared/ 里关于退款的规则                           │
│ │ ▼ shared/││                                                                 │
│ │  ├ 规范.md││  🤖 [✓ 查阅了 1 份文件]                                          │
│ │ ──────── ││     根据 shared/规范.md，我们的退款规则是...                    │
│ │ 📂 工作区││                                                                 │
│ │ ▼ temp/  ││                                                                 │
│ │ 🔒mailbox││                                                                 │
│ └──────────┘│ ┌─────────────────────────────────────────────────────────────┐ │
│             │ │ 📝 输入消息...                             [📎 上传] [发送] │ │
│             │ └─────────────────────────────────────────────────────────────┘ │
└─────────────┴─────────────────────────────────────────────────────────────────┘
```

## 二、 极简顶部控制条

专注于调试与发布闭环。

*   **左侧**：保留面包屑 `← 首页 | App：my_agent`。
*   **右上角悬浮球 (弱化展示)**：
    *   取消了常驻的繁长文本（如 `Gemini 2.0 Pro Token: ↑1.2k ↓3.5k`）。
    *   取而代之的是右上角一个极其低调的灰色信息小标 `[ℹ️ 运行状态]`，鼠标悬浮 (Hover) 时才弹出：`模型: Gemini 2.0 Pro | 消耗: 1.2k In, 3.5k Out`。
*   **主操作区（发版闭环）**：
    *   `[👀 草稿与线上对比]`：**常驻次级按钮**。点击后右侧滑出对比抽屉，展示当前草稿的提示词/工具配置与线上正在运行的版本有何差异，做到发布前心中有数。
    *   `[结束调试]`：点击后，中间画布翻转回去，回到刚写的提示词编辑界面。
    *   `[🚀 发布版本]`：主操作按钮。将当前草稿固化为正式新版本，对外生效。
        *   **【不测不发约束】**：如果用户进入调试页后，没有发过哪怕一条测试消息就直接点击此按钮，系统会弹出软性警告：“您尚未在当前沙盒中进行过验证，直接发布可能会影响线上使用者，是否执意发布？”。强制用户对自己的操作负责。

## 三、 工具调用的降噪展示 (拟人化披露)

采用 **渐进披露 (Progressive Disclosure)**，将技术细节平滑融入阅读流：

### 3.1 运行态与结果态
*   **正在调用 (Loading)**：在回复气泡上方，以极其轻微的灰色小字加动效展示。如：`⚙️ 正在调用 Github 搜索...`。
*   **调用完成 (Done)**：文字收缩为一个可以点击的灰色胶囊 Badge (徽章)。如：`[✓ 查阅了 1 份文件]`。

### 3.2 深度技术细节隔离 (JSON Drawer)
调试毕竟是开发者行为，需要看参数。但绝不要破坏阅读流，尤其面对超长文本的返回。
*   开发者**主动点击**聊天气泡上方那个 `[✓ 查阅了 1 份文件]` 的小 Badge。
*   页面**右侧边缘**平滑滑出一个半透明的黑色遮罩抽屉 (Debug Drawer)，里面纯粹展示当前工具调用的入参和出参（格式化后的代码块）：
    *   `Request (入参)` 和 `Response (出参 / 结果摘要)`。
    *   **超长结果处理**：即使工具返回了 5000 行纯文本日志，也只会在这个黑色抽屉的 Response 区域展示，并支持上下滚动，绝对不糊在主聊天屏幕的气泡中。
    *   关闭抽屉，即刻回到纯净的聊天界面。

## 四、 优雅的表单挂起 (Inline Action Card)

将系统拦截表单融入自然对话流中，让 AI 显得更有礼貌和边界感。
*注：此机制作为系统底层安全规范，将同时在【05_调试模式】和【06_正式模式】中统一生效，确保不论是开发者还是最终消费者，都拥有绝对的数据知情与控制权。*

### 4.1 安全审批与批量权限拦截
当工具请求越权或敏感操作（如重写受保护的文件、调用高危 MCP 接口），系统截停。
*   **表现形式**：AI 会发出一条带有系统背景色的自然语言消息：
    > 🤖 “因为安全限制，我需要您的授权才能执行以下写入操作：”
*   **内联审批卡片 (Inline Action Card)**：
    *   **带复选框的批量授权**：为避免多次弹框打断心流，如果 AI 计划同时修改多个文件，系统会将其合并为一个内联卡片。
    *   卡片内部罗列待操作的清单（如 `☑ workspace/a.txt`，`☑ workspace/b.txt`）。
    *   用户可以自由勾选/取消勾选具体允许的文件。
    *   卡片底部提供统一个 `[拒绝授权]` 与 `[✅ 确认选中授权]` 按钮。
*   点击后，该表单状态永久固化为不可更改（如显示 `已允许 2 项操作`），成为清晰的审批流水记录，对话随之继续。

### 4.2 轮次耗尽 / Token 压缩
*   遇到最大思考步数：
    > 🤖 “我已经思考了很久（达到20步上限），是否需要我继续深挖？”
    > `[就此打住]` `[继续思考]`
*   遇到 Token 触顶边缘：
    系统在对话中间插入一行极细的灰色分割线，中间带字：`✂️ 系统已自动压缩早期上下文以释放记忆空间`。**不阻塞用户，不要求强制点击**。

## 五、 全局异常状态与容错机制 (Graceful Error Handling)

无论是底层服务异常，还是配置过期，系统都遵循“就地提示、体面降级”的原则。针对 App 自身的配置状态，分为以下 4 种核心提示规则：

### 5.1 异常类型与拦截规则总览

| 异常类型 | 顶部提示条文案 | 是否阻断对话 | 应对逻辑 |
| :--- | :--- | :--- | :--- |
| **1. 工具即将过期** (status=2) | `⚠️ 部分工具即将过期，App 正常运行。` | **否** | 纯预警。工具照常可用，提醒开发者关注。 |
| **2. 工具已异常** (失效/不兼容等) | `⚠️ 部分工具暂不可用，请返回编辑页面处理。` | **否** | 对话正常进行。受影响的工具调用时会失败，由大模型自行尝试处理或告知最终使用者。 |
| **3. 主模型失效** | `⚠️ 当前主模型已失效，请结束调试重新选择模型。` | **是** | 输入框被禁用，历史对话降级为只读状态。 |
| **4. 平台分组失效** | `⚠️ 当前底层平台已停止服务，请结束调试切换平台。` | **是** | 输入框被禁用，历史对话降级为只读状态。 |

*(注：所有全局提示条均悬浮显示在顶部导航栏下方，采用醒目的黄色/橙色背景。)*

### 5.3 运行时底层报错隔离 (JSON Drawer)
如果遇到的是接口网络不通等硬核报错：
*   **气泡展示**：AI 头像回复一条红色提醒边框的简短消息，并提供 `[查看错误详情]` 按钮。
*   **详情隔离**：点击后，从屏幕右侧滑出抽屉，将 `AuthenticationError...` 等长篇硬核代码堆栈丢在里面，绝对不污染主要的聊天画布。

## 六、 交互与文件投递

保持 C 端沟通最常用的习惯，但与左侧的独立文件系统进行无缝绑定。

### 6.1 输入框结构
*   **底部固定**：`📝 输入消息... [📎 上传] [发送]`。
*   **直接上传**：点击 `[📎 上传]` 或直接拖拽桌面文件进入输入框，文件会自动推入左侧沙盒的 `assets/` 目录下，并在输入框上方生成一个标签：
    `📄 季度报告.pdf ✕`

### 6.2 沙盒反向投递 (Mention)
因为这往往是一个文件处理工具，使用者不用费力去打字说“帮我处理 workspace 下面的 a.txt”。
*   可以直接用鼠标将左侧文件树里的文件，**拖拽**到右侧聊天输入框里。
*   输入框里会自动生成上述的 `📄 workspace/a.txt ✕` 引用标签，按下发送键，AI 就能心领神会去处理这个具体路径的文件。
