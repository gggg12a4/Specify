# 用户修改记录

- 1. 从模板创建：后台一键克隆该模板的所有预设配置，直接生成新实例并进入开发页。该内容应该是需要进入create页面的。
- 2. 空间隔离（文件系统）：分为开发者共享资料库 (shared/) 和最终用户的独立沙盒 (workspace/, temp/, memory/, assets/,  mailbox/)。这部分内容需要和老板沟通是否需要这些以及用户上传存储到哪里？
- 3. Agent失效阻断机制及运行态降级：该内容需要只能在引用方取消引用；平台功能失效只能关闭然后置灰。另外此处需要注意：修复之后或者更换模型之后的对话记录存储，是需要清空还是说可以继续对话（需确认）。
- 4. 不测不发约束：不经过调试对话验证，直接发布会有软性警告。这个内容需要补充版本的需求交互以及真实确切的数据流转是怎么样的。
